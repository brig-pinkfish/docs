## charting_deploy_chart

Deploy a pre-built Chart.js visualization to a shareable URL.

WHEN TO USE: User explicitly wants a hosted/shareable chart link (e.g., "deploy this chart", "give me a URL", "host this visualization").

NOT FOR: Creating charts - that's done in code-block nodes using guidance from the system prompt. This tool only handles deployment.

REQUIRES: collectionId - filestorage collection to store the deployed chart (use &#123;&#123;resource.collectionName&#125;&#125;).

DATA-DRIVEN CHARTS: If your HTML contains &#123;&#123;DATA_URL&#125;&#125; placeholder, provide sdk_file_id or filestorage_id to resolve it.

**Parameters:**

| Parameter | Type | Required | Default | Description |
| --- | --- | --- | --- | --- |
| `html_content` | string | Yes | — | Required: The Chart.js HTML code to deploy. Include &#123;&#123;DATA_URL&#125;&#125; placeholder if your chart needs to fetch external data. |
| `collectionId` | string | Yes | — | REQUIRED: Collection ID to store the deployed chart file. In workflows, declare a filestorage collection resource in WORKFLOW_RESOURCES (e.g., myCharts: &#123; type: "collection", collectionType: "filestorage" &#125;), bind it via workflow_bind_resources, then use &#123;&#123;resource.myCharts&#125;&#125; here. |
| `sdk_file_id` | string | No | — | Supply data to your chart: File ID from pf.files.writeFile() to resolve for &#123;&#123;DATA_URL&#125;&#125;. Format: automationId/runId/nodeId/filename. NOTE: The resolved S3 URL expires after approximately 15 minutes, so the deployed HTML should fetch the data immediately when loaded. |
| `filestorage_id` | string | No | — | Supply data to your chart (preferred): File ID from filestorage MCP tool response to resolve for &#123;&#123;DATA_URL&#125;&#125;. Expiration depends on how the file was uploaded (can range from 1 minute to public/never). |
| `output_filename` | string | No | `"chart.html"` | Optional: Name for the output chart file (default: "chart.html") |
| `file_links_expire_in_days` | string | No | `7` | Number of days before the chart URL expires (1-30), or "never" for a public URL with no expiration (default: 7) |
