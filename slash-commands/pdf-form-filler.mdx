## What can you do with it?

Fill PDF forms programmatically with data. Supports text fields, checkboxes, and dropdowns with intelligent field matching and multiple fallback strategies.

## How to use it?

### Basic Command Structure
```
/pdf-form-filler
pdf file: the PDF form to fill
field data: the data to fill into the form
```

### Parameters
- **pdf file**: URL or path to the PDF form
- **field data**: Key-value pairs of field names and values
- **checkbox values** (optional): Boolean or string values for checkboxes
- **dropdown selections** (optional): Values to select in dropdown fields

### Response Format
Returns a base64-encoded filled PDF with metadata including filename and mime type.

## Examples

### Basic Usage
Fill a simple form:
```
/pdf-form-filler
pdf file: application_form.pdf
field data: {"name": "John Doe", "email": "john@example.com", "date": "2024-01-15"}
```

### Advanced Usage
Fill complex form with checkboxes:
```
/pdf-form-filler
pdf file: registration_form.pdf
field data: {"firstName": "Jane", "lastName": "Smith", "agreeToTerms": true, "newsletter": "checked"}
```

### Specific Use Case
Fill tax form with dropdowns:
```
/pdf-form-filler
pdf file: tax_form.pdf
field data: {"taxYear": "2023", "filingStatus": "Single", "income": "50000", "deductions": "Standard"}
```

### Notes
The skill uses intelligent field matching with fallbacks:
- Direct field name matching
- Similar name pattern matching
- Positional fallback as last resort
- Multiple checkbox methods (.check(), setValue variations)
- Dropdown option matching with fuzzy search
