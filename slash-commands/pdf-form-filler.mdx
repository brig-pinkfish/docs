## What can you do with it?

Fill PDF forms programmatically with data. Supports text fields, checkboxes, and dropdowns with intelligent field matching and multiple fallback strategies. Perfect for automating form submissions, batch processing applications, and streamlining document workflows.

## How to use it?

### Basic Command Structure
```
/pdf-form-filler [pdf-file] [field-data]
```

### Parameters

**Required:**

- `pdf file` - URL or path to the PDF form

- `field data` - Key-value pairs of field names and values

**Optional:**

- `checkbox values` - Boolean or string values for checkboxes

- `dropdown selections` - Values to select in dropdown fields

### Response Format

The command returns:
```json
{
  "pdf": "base64-encoded filled PDF",
  "metadata": {
    "filename": "filled_form.pdf",
    "mime_type": "application/pdf",
    "fields_filled": "number of fields populated"
  }
}
```

## Examples

### Basic Usage
```
/pdf-form-filler
pdf file: application_form.pdf
field data: {"name": "John Doe", "email": "john@example.com", "date": "2024-01-15"}
```
Fill a simple form with basic text field data.

### Advanced Usage
```
/pdf-form-filler
pdf file: registration_form.pdf
field data: {"firstName": "Jane", "lastName": "Smith", "agreeToTerms": true, "newsletter": "checked"}
```
Fill complex form including checkbox fields with various value formats.

### Specific Use Case
```
/pdf-form-filler
pdf file: tax_form.pdf
field data: {"taxYear": "2023", "filingStatus": "Single", "income": "50000", "deductions": "Standard"}
```
Fill tax form with dropdown selections and numeric fields.

### Notes

Uses intelligent field matching with fallbacks: direct field name matching, similar name pattern matching, and positional fallback as last resort. Supports multiple checkbox methods (.check(), setValue variations). Dropdown option matching includes fuzzy search for best match.
