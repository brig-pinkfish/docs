The `/polygon-io` command enables you to access comprehensive financial market data through the Polygon.io API. Perfect for:

- Stock market data and analysis
- Options contract information
- Real-time market snapshots
- Historical price data
- Financial market research

## Basic Usage

Use the command to access financial data:

```
/polygon-io get AAPL stock data for the last week
```

```
/polygon-io find options contracts for TSLA expiring next month
```

```
/polygon-io get market snapshot for major tech stocks
```

## Key Features

### Stock Data
- OHLCV (Open, High, Low, Close, Volume) data

### Market Information

## Stock Data Operations

### Get Aggregates (OHLCV Data)
```javascript
// GET aggs/ticker/{ticker}/range/{multiplier}/{timespan}/{from}/{to}
const url = POLYGON_URL + `aggs/ticker/AAPL/range/1/day/2023-01-09/2023-01-13`;
```

### Aggregates Response
```javascript
{
  "ticker": "AAPL",
  "queryCount": 5,
  "resultsCount": 5,
  "adjusted": true,
  "results": [
    {
      "v": 76532548,         // volume
      "vw": 130.8995,        // volume-weighted average price
      "o": 130.465,          // open price
      "c": 130.73,           // close price
      "h": 132.4899,         // high price
      "l": 129.89,           // low price
      "t": 1673298000000,    // timestamp (Unix time in milliseconds)
      "n": 657212            // number of trades
    }
  ],
  "status": "OK",
  "count": 5
}
```

### Supported Timeframes
- **minute**: Minute-level data
- **hour**: Hourly aggregates
- **day**: Daily data
- **week**: Weekly aggregates
- **month**: Monthly data
- **quarter**: Quarterly data
- **year**: Annual data

## Data Format Requirements

### Date Format
- **Required Format**: YYYY-MM-DD (with hyphens)
- **Example**: 2023-01-09
- **Invalid**: 2023/01/09 or 20230109

### Supported Resolutions
- minute
- hour
- day
- week
- month
- quarter
- year

### Default Parameters
- **multiplier**: 1
- **timespan**: day
- **from**: current date
- **to**: tomorrow date

## Connection Requirements

### PinkConnect Proxy
- Uses PC_BASE_URL + v3/ endpoint
- Equivalent to https://api.polygon.io/v3/
- Requires Polygon.io connection ID

### Paragon Proxy
- Uses PROXY_URL + custom/{integrationId}/v2/
- Alternative authentication method
- Different endpoint structure

## Best Practices

1. **Date Handling**
   - Always use YYYY-MM-DD format with hyphens
   - Validate date ranges before requests
   - Consider market holidays and weekends
   - Use appropriate timeframes for analysis

2. **Data Analysis**
   - Combine multiple data points for insights
   - Use volume-weighted averages for accuracy
   - Consider after-hours and pre-market data
   - Implement proper error handling

3. **Performance**
   - Cache frequently accessed data
   - Use appropriate date ranges
   - Batch requests when possible
   - Monitor API rate limits

4. **Options Trading**
   - Verify expiration dates
   - Check strike price ranges
   - Understand exercise styles
   - Monitor contract specifications

## Data Interpretation

### Price Data
- **o**: Opening price
- **h**: Highest price
- **l**: Lowest price
- **c**: Closing price
- **v**: Volume
- **vw**: Volume-weighted average price
- **n**: Number of trades

### Trade Data
- **p**: Price
- **s**: Size (shares)
- **t**: Timestamp
- **c**: Conditions array

## Tips
- Always use YYYY-MM-DD date format with hyphens (mandatory)
- Supported resolutions are limited to minute|hour|day|week|month|quarter|year
- Default parameters include multiplier=1, timespan=day
- Use volume-weighted average price (vw) for more accurate analysis
- Consider market hours when interpreting real-time data
- Cache company information as it changes infrequently
- Monitor API rate limits to avoid service interruptions
