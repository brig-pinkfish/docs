**Embedded Services** are the MCP servers that ship with Pinkfish. Unlike [Connections](/essentials/api-connections), they don't require authentication or setup — every agent in your org can call them the moment it starts. They cover the capabilities most workflows reach for first: searching the web, running code, generating charts, processing images and documents, sending email, and reading from your own resources.

Find them under **Tools → Embedded Services**.

![Embedded Services list with descriptions and the test action on each row](/images/embedded-services/embedded-services-overview.png)

## What's Available

Embedded services are grouped by what they do. The exact catalog in your org depends on what's enabled for your plan; common services include:

| Category | Services |
|----------|----------|
| Web | Web Search, Web Scraping, HTTP Utilities |
| Content | Charting, Image Processing, Audio Processing, Video Processing, Document Processing |
| Data | Datastore (Structured), Datastore (Unstructured), File Storage, Knowledge Base, Queue Management |
| Agent runtime | Code Execution, Browser Automation, Email, Vault, Agent Skills |

Each row on the list shows the service's display name, a short description of what it does, and a **Test MCP tools** action.

For the full reference of every tool each service exposes, see the [Embedded MCP Servers](/api-reference/mcp-servers/embedded/overview) section.

## Testing a Service

Click the flask icon on any row to open the **Test {Service} MCP** dialog. The left panel lists every tool the service provides; the right panel renders a form generated from the tool's input schema.

![Test Web-Search MCP dialog — tools on the left, parameter form with Execute Test on the right](/images/embedded-services/embedded-services-test.png)

Typical flow:

1. Select a tool from the left.
2. Fill in the required parameters and any **Optional Fields** you want to set.
3. Click **Execute Test**.
4. Inspect the response in the result panel.

This is the fastest way to learn what a tool expects and to sanity-check a call before building it into an agent or workflow.

## Using Embedded Services in Agents and Workflows

Embedded services are available to every agent without any wiring. Reference them by name in an agent's tool list, or invoke them from a workflow step. Because they're always on, you don't need to pick a connection at call time — just pick the tool.

For details on any individual service, open the [Embedded MCP Servers reference](/api-reference/mcp-servers/embedded/overview) and choose the service you're interested in.

## Notes

- Embedded Services require the Builder role to view from the Tools page.
- The catalog is curated by Pinkfish; new services appear automatically as they're released.
- Some services (like Vault and Knowledge Base) read from your own [resources](/essentials/tools#resources) — test calls will operate on real data, so use them with the same caution you would in a workflow.
