The **Tools** page is where you manage everything your agents and workflows can reach out to: third-party apps you've connected, Pinkfish's built-in services, custom MCP servers registered by your team, structured and unstructured data resources, and authenticated browser sessions. Anything an agent can call, you configure here.

Navigate to **Tools** from the top nav. The page opens on **Connections** by default.

![Tools page showing the Connections list with the sidebar on the left](/images/tools/tools-overview.png)

## MCPs

Every tool an agent can call is exposed as an MCP (Model Context Protocol) server. The MCPs section has three pages.

### Connections

Your list of connected third-party apps — Salesforce, Slack, Google Sheets, HubSpot, and so on. Each row shows the app logo, connection name, category, last update time, and status indicator.

Row actions:

- **Test MCP tools** (flask icon) — open a dialog that lists every tool the app exposes, lets you fill in parameters as JSON, run the tool, and inspect the live response.
- **Reconnect** (refresh icon) — available for OAuth-based apps when the session has expired.
- **Edit** — rename the connection or adjust its configuration.
- **Share** — grant view or edit access to specific team members (admins only).
- **Delete** — remove the connection (write permission required).

Click **Add Connection** in the top right to launch the integration picker.

![Add Connection — integration picker with Custom Apps and Available Apps grid](/images/tools/tools-add-connection.png)

Pick an app, authenticate, and save. The new connection joins the list immediately. If the app you want isn't there, you can build your own — see [API Connections](/essentials/api-connections).

### Testing a connection's tools

Click the flask icon on any row that supports MCP tools. The **{App} Tools** dialog opens with the full tool catalog on the left and a parameter form on the right. Select a tool, fill in the inputs, and run it without ever leaving the Tools page.

![ActiveCampaign Tools dialog showing the Available Tools list](/images/tools/tools-test-modal.png)

This is the fastest way to verify a connection is healthy and to learn what a tool expects before you wire it into an agent or workflow.

### Embedded Services

Pinkfish's built-in MCP servers — web search, charting, email, code execution, browser automation, image and video processing, knowledge base retrieval, and more. These don't require a connection; they're always available to agents in your org. See [Embedded Services](/essentials/embedded-services).

### Custom MCP Servers

Remote MCP servers your org has registered with Pinkfish. Each custom server contributes its own tools into the agent catalog. See [Custom MCP Servers](/essentials/custom-mcp-servers).

## Resources

The Resources section is where agents read and write data. Each resource is organized into **collections** — you create a collection, set its schema or configuration, and then reference it from agents and workflows. Sharing rules follow the same view/edit/admin model as connections.

  <Card title="File Store" icon="folder" href="/essentials/filestore">

  </Card>
  <Card title="Knowledge Base" icon="brain" href="/essentials/knowledgebase">

To create a resource, click the section in the sidebar, then use the **Create** button on the list page. Drill into a collection to add rows, upload files, index documents, or store secrets.

## Browser

### Browser Login

When a service doesn't expose an API, Pinkfish can automate it through an authenticated browser session. **Browser Logins** is where you create and manage those saved sessions. Once a login is stored, your workflows can drive the `/scraper` and `/browser session` skills against it. See [Browser Logins](/essentials/browser-logins).

## Permissions and Sharing

Every item on the Tools page has access controls:

| Action | Required permission |
|--------|---------------------|
| View and use | Read |
| Edit, rename, reconfigure | Write |
| Share with others | Admin |
| Delete | Write |

Shared resources show a small "Shared" indicator next to the name. Use the **Share** action on any connection or resource to invite specific teammates.

## Notes

- The **Embedded Services**, **Database**, **File Store**, **Vault**, and **Browser Login** sections require the Builder role.
- Vault additionally requires an eligible plan.
- Connection tools only appear in the **Test MCP tools** dialog once the connection is established and at least one MCP server has been generated for the app.
