Connect your agents and Coworker to Slack so your team can get work done from the tool they already use. Send a DM to the Pinkfish Slack app from your laptop or phone and an agent replies in the thread — with a written answer, any files it produced, and links to rendered web-app visualizations for data that deserves more than text.

## How It Works

1. **Install the Pinkfish Slack app** into your workspace via OAuth. You can connect one agent at a time, or enable Slack for the entire organization.
2. **Open a DM** with the Pinkfish app in Slack.
3. **Send a message.** Pinkfish identifies you by the email on your Slack profile and matches it to your Pinkfish account.
4. **An agent responds in the thread.** If the DM is wired to a specific agent, that agent answers. Otherwise Pinkfish picks the best match from the agents available in the workspace and asks you to confirm.
5. **Outputs flow back.** Short text answers appear as Slack messages. Files the agent produces are uploaded to the thread. Rendered HTML artifacts (dashboards, charts, interactive views) come through as links that open the web app in your browser.

## Enabling Slack for an Agent

Open the agent you want to reach from Slack and go to the **Channels** section of the agent editor.

![Agent editor showing the Slack channel card with a toggle and description](/images/slack-channel/slack-channel-toggle.png)

1. Toggle **Slack** on.
2. A browser popup opens and redirects you to Slack to authorize the Pinkfish app in your workspace.
3. Approve the requested permissions. Slack returns you to Pinkfish, and the agent is now reachable from a DM in that workspace.
4. The Pinkfish app sends you a welcome DM confirming the connection.

To disconnect, toggle **Slack** off on the same card. This removes the channel binding for the agent; it does not uninstall the Slack app from the workspace.

### Organization-wide Slack

If your organization has Slack enabled globally, every published agent is reachable from Slack by default. The per-agent Slack toggle flips to an **opt-out** control: leave it on to keep the agent available in Slack, or turn it off to hide just that agent.

![Settings → Agents page showing the Channels section with a Slack toggle that enables all published agents in the org's Slack workspace](/images/slack-channel/slack-channel-org-settings.png)

Admins turn this on under **Settings → Agents → Channels**. It's useful when most of your team works from Slack and you'd rather whitelist exceptions than opt each agent in one by one.

## Using Coworker from Slack

You don't need to pre-select an agent to start a conversation. Send a plain-language request to the Pinkfish app and Coworker behavior kicks in: Pinkfish routes the message to the most relevant agent in your workspace, or answers directly if no specialized agent is needed.

A typical flow:

- From your phone, DM the Pinkfish app: *"Give me a 360 on Global Media."*
- Pinkfish finds a matching agent (for example, a Customer 360° agent) and runs it.
- The agent replies in the thread with a written summary and attaches a link to a rendered web app with the full visual breakdown — charts, contacts, recent activity, open deals — ready to open in your browser.

Because every DM with the Pinkfish app is a thread, each conversation stays scoped and easy to come back to later.

## Supported Artifact Types

| Artifact type | How it appears in Slack |
|---------------|-------------------------|
| Short text answers | Posted directly as Slack messages |
| Generated files (images, PDFs, CSVs, etc.) | Uploaded to the thread as Slack files |
| Rendered HTML web apps | Posted as shareable links that open the web app in your browser |

Web-app artifacts stay interactive — they are not flattened to images — so charts, filters, and other controls work when you open the link.

## Notes

- The Pinkfish Slack app authenticates you using the email on your Slack profile. Make sure that email matches the one on your Pinkfish account.
- An agent connects to a single Slack workspace per organization.
- Conversations happen in DMs with the Pinkfish app; each new thread starts a new chat.
- The Slack Channel is a separate feature from the [Slack MCP server](/slash-commands/slack), which lets agents act on Slack as a tool (posting to channels, reading messages, and so on). The two can be used together.
- For other ways to start a workflow or agent, see [Triggers](/essentials/triggers). For Coworker inside the Pinkfish app, see [Coworker](/essentials/coworker).
