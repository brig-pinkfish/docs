**Custom MCP Servers** let your organization plug external MCP servers into Pinkfish. Once registered, every tool the server exposes shows up in the agent tool catalog and in the workflow builder's slash menu — right next to Pinkfish's [embedded services](/essentials/embedded-services) and third-party [connections](/essentials/api-connections).

Find them under **Tools → Custom MCP Servers**.

![Custom MCP Servers list with empty state](/images/custom-mcp-servers/custom-mcp-servers-overview.png)

## When to Register a Custom MCP Server

Reach for a custom MCP server when:

- You've built or deployed an MCP server that wraps an internal system (a proprietary API, a data warehouse, a service mesh) and you want Pinkfish agents to use it.
- A vendor ships an official remote MCP server and you'd like its tools available without building a new integration.
- You're extending Pinkfish with specialized capabilities that aren't covered by the embedded catalog.

For connections to hosted SaaS apps that expose a normal REST API, use [API Connections](/essentials/api-connections) instead — those are optimized for OAuth and API-key auth flows.

## What Each Row Shows

Every registered server appears as a row with:

- **Name and description** — the display label and short summary you provided at registration.
- **More menu** — actions available for that server.

### Row actions

| Action | Description | Required permission |
|--------|-------------|---------------------|
| View Details | Open a side panel listing the server ID, service key, and every tool the server exposes. | Any member |
| Delete | Remove the server from the org. Agents and workflows that reference its tools will need to be updated. | Write |

## Server Details

Click **View Details** on the More menu to open the details panel. It contains:

- **Server ID** — the internal identifier for the server.
- **Service Key** — the stable key your agents reference when calling this server's tools.
- **Tools** — a count badge plus a card for every tool, showing the tool name as a monospace badge and a short description pulled from the server's own schema.

If a tool doesn't have a description, Pinkfish shows `No description available` rather than hiding the row — so you always know what's registered.

## Deleting a Server

From the More menu, choose **Delete**. A confirmation dialog asks you to confirm — this action cannot be undone. On success a toast appears with the deleted server's name. Any agent or workflow that still references the server's tools will start failing at call time, so audit usage before removing anything in active use.

## Using Custom MCP Servers

Once a server is registered, its tools behave exactly like every other tool in Pinkfish:

- Agents can be configured to use them.
- Workflow steps can invoke them through the slash menu.
- The tool name follows the registered service key as a prefix, which keeps names unique across servers.

## Notes

- Registering a new custom MCP server is handled by your org admin through the platform's MCP registration flow — reach out to support@pinkfish.ai if you need a server added and you don't have admin access.
- Empty state text is `No custom MCP servers available` — if you expected a server to be listed and it isn't, confirm it was registered against the current org.
