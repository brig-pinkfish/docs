Saved Actions let Pinkfish remember code it has already written and run for you, so the next time you send a similar prompt, the same working solution can be reused instantly. This cuts down on redundant code generation, makes repeated tasks faster, and keeps known-good patterns at your fingertips.

## How It Works

When an agent writes and successfully executes code to answer one of your prompts, you can save that prompt + code pair by giving it a thumbs-up. Pinkfish stores the pair in a vector index scoped to you, your organization, and (optionally) a shared global pool.

The next time you send a prompt, Pinkfish performs a similarity search against these saved pairs. If a strong enough match is found, the matching code is suggested to the agent as a starting point — the agent can run it as-is, adapt it, or ignore it and generate fresh code.

Matching is cascading: your personal saved actions take priority, then your organization's, and finally the global pool shared across all organizations.

## Saving an Action

After an agent finishes a response that included a successful code execution, a thumbs-up and thumbs-down button appear below the message.

- **Thumbs up** — Saves the prompt and the executed code as a new saved action.
- **Thumbs down** — Marks the response as unhelpful. If the response came from a previously saved action, the saved action is deleted.

You'll see a confirmation message (for example, "Saved action created" or "Saved action deleted") next to the buttons after you click.

<Tip>
  The prompt saved is the original request you sent — not any intermediate tool calls — and the code saved is the last successful code execution from that response.
</Tip>

## Replaying a Saved Action

### Automatic replay

Just send a prompt. If it closely matches a saved action you (or your organization, or the global pool) have created, Pinkfish injects the matching code as a suggestion into the conversation. The agent can use it directly, which typically produces a faster and more consistent result.

When the response comes from a saved action match, the thumbs-up button changes to **"This result was correct"** — clicking it confirms the match worked, and clicking thumbs-down removes the saved action that was matched.

### Inserting manually with `/`

In any Coworker chat input, press `/` on an empty line to open the Saved Actions picker.

![Saved Actions picker opened from the chat input](/images/saved-actions/slash-command.png)

- Type to search through your saved actions by prompt text.
- Click an entry to insert its prompt into the chat input so you can send it (or edit it first).
- Click the trash icon on a row to delete that saved action.

## Managing Your Saved Actions

Navigate to **Settings → Saved Actions** (under the **PERSONAL** section) to see and manage everything you've saved.

![Personal Saved Actions settings page](/images/saved-actions/saved-actions-settings.png)

From this page you can:

- **Enable Saved Actions** — toggle the entire feature on or off for your account. When disabled, the thumbs-up controls and the `/` slash command in chat are hidden, and no automatic matching is performed against your prompts.
- **Your Saved Actions** — browse the full list, with the prompt preview and creation date for each entry.
- **Delete** — remove any individual saved action with the trash icon.

New saved actions appear here as soon as you thumbs-up a response in chat.

## Organization-Level Controls

Organization administrators have an additional settings page under **Settings → Saved Actions** (in the **ORGANIZATION** section).

![Organization Saved Actions settings page](/images/saved-actions/org-saved-actions-settings.png)

- **Opt out of global saved actions pool** — when enabled, prompt matching only considers saved actions created by members of your organization. The global pool shared across all Pinkfish organizations is excluded.

Use this toggle when your organization wants stricter isolation — for example, to make sure prompts are only answered with code that has been reviewed and approved internally.

## Scope and Privacy

Every saved action is tagged with the user and organization that created it. Matching is performed in this order:

1. **Personal** — your own saved actions (always searched).
2. **Organization** — saved actions created by any member of your organization (always searched).
3. **Global** — saved actions created by any Pinkfish user, unless your organization has opted out of the global pool.

A saved action is only ever visible to you in the Settings UI — other users can't browse the list, even if their prompts match and reuse your code.

## Notes

- Saved Actions are enabled by default for all accounts. Administrators can't disable the feature org-wide, but individual users can turn it off from **Settings → Saved Actions**.
- There's no expiry: saved actions live until you delete them, either from the Settings page, from the `/` slash command picker, or by thumbs-down on a cached response.
- Only successful code executions trigger the save-able state — if an agent's response didn't run code, the thumbs-up control won't appear.
