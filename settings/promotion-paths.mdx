**Promotion Paths** let you define named environments — typically **Dev**, **Stage**, **Prod** — and the rules for moving a workflow release between them.

Open **Settings → Promotion Paths** (admin only).

## What a promotion path does

A promotion path is the policy that governs:

- Which workflows are allowed in which environment.
- Who can promote a workflow to the next environment.
- What connections, resources, and API credentials each environment uses.
- Required approvals or checks before promotion.

## Typical setup

<Steps>
  <Step title="Define environments">
    Create Dev, Stage, and Prod. Each gets its own set of resources (datastores, file stores, vaults) and connections.
  </Step>
  <Step title="Assign approvers">
    Decide who can promote at each boundary. Dev → Stage might be self-service; Stage → Prod might require an admin approval.
  </Step>
  <Step title="Promote">
    From the [Workflow Library](/workflows/library), click **Promote** on a released workflow. Pick the destination environment. Pinkfish validates that all dependencies exist in the target env before the release lands.
  </Step>
</Steps>
