**Models** is where org admins choose the large-language-model defaults that power your agents and workflows. Pinkfish ships with sane defaults; this page is for teams that want to pin a specific model, prefer a specific vendor, or roll their own via [BYOM](/organization/byom).

Open **Settings → Models** (admin only).

## What you can configure

- **Default model for Agents** — the model new agents inherit unless overridden per-agent.
- **Default model for Workflows** — the model Agent Mode uses when reasoning through a workflow build.
- **Allowed models** — the subset of models your org is willing to run on (for cost, data-residency, or compliance reasons).
- **Fallback order** — if the primary model is unavailable, which model Pinkfish tries next.

## Supported vendors

Out of the box: Anthropic Claude, OpenAI, Google Gemini, Groq, Perplexity. Private hosted and BYOM models are configured through [BYOM](/organization/byom).

## Per-agent override

Any agent can pin its own model from the [Instructions tab](/agents/instructions) of Agent Builder — that overrides the org default for that single agent.
