Pinkfish has **four product surfaces**. Understanding what each is for makes the rest of the docs easy to navigate.

## The four features

<CardGroup cols={2}>
  <Card title="Coworker" icon="handshake" href="/coworker/overview">
    A chat where you talk to Pinkfish. It can answer questions, run your agents, kick off workflows, and save repeatable actions. Use it when the work is ad-hoc.
  </Card>
  <Card title="Workflows" icon="diagram-project" href="/workflows/overview">
    Visual automations. You describe the process, Agent Mode lays out the nodes, you run it manually or on a trigger. Use when the work is repeatable and predictable.
  </Card>
  <Card title="Agents" icon="robot" href="/agents/introduction">
    Configured AI assistants. Each has instructions, a set of tools, access to resources, and channels (Slack, Teams, public URL, Coworker). Use when you want a reusable, shareable assistant for a role.
  </Card>
  <Card title="Tools" icon="toolbox" href="/tools/overview">
    Everything your agents and workflows can *do* — integrations (MCPs), skills, storage (databases, files, knowledge bases, secrets), and browser login. Use this section when you're wiring up capability.
  </Card>
</CardGroup>

## How they fit together

- An **Agent** uses **Tools** to take actions, reads from **Storage** (under Tools), and shows up in **Coworker** or a channel.
- A **Workflow** uses **Tools** too, and can be called *by* an agent, triggered by an event, or run from **Coworker**.
- **Coworker** is where everything becomes reachable in conversation.

## Supporting vocabulary

| Term | What it means |
| --- | --- |
| **MCP** | Model Context Protocol server — a bundle of tools an agent can call. Pinkfish ships many ([Built-in Tools](/api-reference/mcp-servers/embedded/overview)) and connects to hundreds of apps ([Integrations](/api-reference/mcp-servers/application/overview)). |
| **Skill** | A named, reusable recipe for an MCP tool — a shorter way to invoke common patterns. See [Skills](/tools/skills). |
| **Resource** | Data the agent/workflow can access: a Datastore (structured), Filestore (files), Knowledge Base (RAG), or Vault (secrets). See [Storage](/resources/database). |
| **Channel** | Where an agent lives — Coworker, Slack, Microsoft Teams, or a public URL. See [Agent Channels](/agents/channels). |
| **Trigger** | What kicks off a workflow — event, email, webhook, API call, schedule, or manual run. See [Triggers](/triggers/overview). |
| **Release** | An immutable published version of a workflow or agent. Draft → Release. See [Releases](/workflows/releases). |
