A **trigger** is what causes a workflow to run. Pinkfish gives you several ways to start one — from ad-hoc clicks in the app to fully automated hooks into external systems.

## Ways to trigger a workflow

<CardGroup cols={2}>
  <Card title="Event Triggers" icon="bell" href="/triggers/event-triggers">
    React instantly to changes in a Datastore, Filestore, or Knowledge Base — no polling.
  </Card>
  <Card title="Email Triggers" icon="envelope" href="/triggers/email-triggers">
    Send an email to a dedicated address and fire a workflow with the subject, body, and attachments.
  </Card>
  <Card title="Google Triggers" icon="google" href="/how-to/google-triggers">
    Kick off workflows from Google Forms, Sheets, Drive, and Calendar.
  </Card>
  <Card title="SMS Triggers" icon="mobile" href="/how-to/sms-trigger">
    Run a workflow when an SMS is received at a Pinkfish-managed number.
  </Card>
  <Card title="Webhooks" icon="plug" href="/api-reference/endpoint/webhook-trigger">
    POST to a signed URL from any service to trigger a workflow with a custom payload.
  </Card>
  <Card title="API" icon="code" href="/api-reference/endpoint/trigger-an-automation">
    Trigger a workflow programmatically with an API key and a JSON body.
  </Card>
  <Card title="Manual Run" icon="play" href="/agentic-editor/running">
    Run from the Agent Mode editor — useful while building and debugging.
  </Card>
  <Card title="From an Agent" icon="robot" href="/agents/tools">
    Attach a workflow as a tool and let an agent call it as part of a conversation.
  </Card>
  <Card title="From Coworker" icon="handshake" href="/coworker/overview">
    Ask Coworker to run a workflow by name — it picks the right inputs from chat.
  </Card>
  <Card title="Scheduled (Trigger Node)" icon="clock" href="/agentic-editor/triggers">
    Use the Triggers node on the canvas to configure time-based or recurring runs.
  </Card>
</CardGroup>

## Which one should I use?

| If you want to... | Use |
|-----|-----|
| React to data changes in Pinkfish | **Event Triggers** |
| Let someone email in a request | **Email Triggers** |
| Collect form responses from the outside | **Google Triggers** |
| Take inbound text messages | **SMS Triggers** |
| Connect a third-party system you control | **Webhooks** |
| Call from your own code | **API** |
| Debug while building | **Manual Run** |
| Let an agent decide to run it | **From an Agent** |
| Drive it from a chat with a teammate | **From Coworker** |
| Run it on a schedule | **Trigger Node** |

Pick the trigger that matches where the work naturally starts. You can combine them — the same workflow can be triggered by an event, a webhook, and a manual run without any change to its logic.
